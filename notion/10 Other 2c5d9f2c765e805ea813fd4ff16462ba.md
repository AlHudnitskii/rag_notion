### Data Lake

[**Data Lake (озеро данных)**](https://www.google.com/search?q=Data+Lake+%28%D0%BE%D0%B7%D0%B5%D1%80%D0%BE+%D0%B4%D0%B0%D0%BD%D0%BD%D1%8B%D1%85%29&client=opera&hs=d33&sca_esv=068c153c500fab67&sxsrf=AE3TifPI4bCa1pgRqapwkM3CPvNFBVxIag%3A1765371656187&ei=CG85af6aC-qD1fIP29TQiQg&ved=2ahUKEwjsyp6dirORAxVnT1UIHUsTDwAQgK4QegQIARAB&uact=5&oq=data+lake+%D1%87%D1%82%D0%BE+%D1%82%D0%B0%D0%BA%D0%BE%D0%B5+%D0%B8+%D0%BF%D1%80%D0%B8%D0%BC%D0%B5%D1%80%D1%8B%3F&gs_lp=Egxnd3Mtd2l6LXNlcnAiLmRhdGEgbGFrZSDRh9GC0L4g0YLQsNC60L7QtSDQuCDQv9GA0LjQvNC10YDRiz8yBRAhGKABSJQoUM4HWLImcAh4AZABAJgB0QGgAdQTqgEGMC4xNS4yuAEDyAEA-AEBmAIOoAKzBsICChAAGLADGNYEGEfCAgQQIxgnwgIEEAAYHsICCBAAGIAEGKIEwgIFEAAY7wXCAgkQIRigARgKGCrCAgcQIRigARgKwgIFECEYnwWYAwCIBgGQBgiSBwM4LjagB49KsgcDMC42uAeQBsIHBjAuMTAuNMgHJYAIAA&sclient=gws-wiz-serp&mstk=AUtExfDSZM0b8TFhiPLHI2t0NYRk1XrU5HaWlBvzmM--mVFDaZDctqJL8CIBIriHu6B2bwkdluZIjzHu_SlaHjZ_aCp6ciQHhKqruVNpI8_EXJa8AaTmpKiwGBamI0fx0ltsncoicMuX_KVco08pn7z3I105iLJ7fHjrqA6fCytwF03xLtTQsCTqoTBIoF51ZH69KcaK3AwzcbDqWxm4d0wecr4PytslZnpdzam8t1b41y7rUDz0yCJRLDUxrURZCNoyVk2l8yoYcKLqE00F6JcKAKgtljnaZfjeM9yJgUQ6Tfij31ZC4QIzGCSeJosthJy5EFpKdoov-UZnqe4Fg3PvLbytylkXUISlqtR_HDEUSALD6GUwZESDyI5qjWrh9AVZshJ4d3F8Rf3TX2Qs_65p2Q&csui=3) — это **централизованное хранилище для огромных объемов данных в их исходном, "сыром" виде (структурированных, полуструктурированных и неструктурированных), без предварительной обработки и обязательной схемы**, что позволяет гибко использовать их для аналитики, машинного обучения и бизнес-интеллекта, когда это потребуется, в отличие от традиционных хранилищ, требующих схемы заранее. Примеры использования включают анализ клиентских данных в телекоме, прогнозирование в нефтегазовой отрасли, автоматизацию диагностики в медицине и персонализацию в розничной торговле.

**Что такое Data Lake?**

- **Централизованное хранилище:** Собирает данные из разных источников (CRM, датчики, соцсети, медиа) в одном месте.
- **"Сырые" данные:** Хранит информацию в оригинальном формате (видео, аудио, тексты, лог-файлы, таблицы) без преобразований.
- **Отсутствие схемы (Schema-on-Read):** Структура данных определяется в момент их извлечения и использования, а не при загрузке.
- **Гибкость и масштабируемость:** Легко адаптируется к растущим объемам данных и различным задачам анализа.

**Примеры использования Data Lake**

- **Телекоммуникации:** Анализ трафика и данных о клиентах для улучшения услуг.
- **Розничная торговля:** Анализ покупательского поведения, управление запасами, персонализация предложений.
- **Медицина:** Интеграция данных из различных медицинских систем для поддержки диагностики и исследований.
- **Нефтегазовая промышленность:** Прогнозирование добычи и оптимизация логистики.
- **Машинное обучение:** «Песочница» для дата-сайентистов, где они могут исследовать «сырые» данные для создания новых моделей.

**Преимущества**

- **Долгосрочное хранение:** Сохраняет данные для будущих, еще не придуманных аналитических задач.
- **Экономичность:** Дешевле хранить большие объемы данных в исходном виде.
- **Всесторонняя аналитика:** Позволяет объединять и анализировать разные типы данных, которые нельзя было бы совместить в классических базах данных.

### Способы обеспечения согласованности и доступности данных в распределенных системах

![image.png](image%20778.png)

- **Кворум (Quorum):** Минимальное количество узлов, которое должно участвовать в операции для ее успешного завершения. Обычно определяется по формуле:
1. **Кворум записи ($W$ - Write):** Минимальное число реплик, на которые должна быть произведена запись.
2. **Кворум чтения ($R$ - Read):** Минимальное число реплик, у которых нужно прочитать данные.
3. Для обеспечения согласованности (Consistency) должно выполняться условие: $W + R > N$, где $N$ — общее число реплик.

### Content Delivery Network (CDN)

**CDN** (**Content Delivery Network** или **Сеть доставки контента**) — это географически распределенная сеть прокси-серверов и их дата-центров (PoP - Point of Presence).

### Назначение:

Основная цель CDN — **ускорить доставку контента** (изображений, видео, скриптов, стилей) конечным пользователям и снизить нагрузку на основной сервер (Origin Server).

### Принцип работы:

1. Контент с основного сервера **кэшируется** на серверах CDN, расположенных ближе к пользователям.
2. Когда пользователь запрашивает контент, CDN автоматически направляет его на ближайший к нему PoP-сервер (Edge Server).
3. Пользователь получает данные быстрее из-за меньшего сетевого расстояния.

### MinIO

**MinIO** — это высокопроизводительное, распределенное **объектное хранилище (Object Storage)** с открытым исходным кодом.

- **Назначение:** Хранение неструктурированных данных (объектов), таких как фотографии, видео, файлы резервных копий, лог-файлы.
- **Совместимость с Amazon S3:** MinIO полностью совместим с API Amazon S3 (Amazon Simple Storage Service), что позволяет легко переносить приложения, разработанные для S3, на собственную инфраструктуру.
- **Ключевые особенности:**
    - **Высокая производительность:** Может читать и писать данные с очень высокой скоростью.
    - **Горизонтальное масштабирование:** Легко добавлять новые узлы для увеличения емкости и производительности.
    - **Erasure Coding:** Использует метод кодирования с исправлением ошибок для защиты данных от сбоев дисков, обеспечивая избыточность.

### S3

Оно предоставляет неограниченное пространство, высокую доступность и масштабируемость, что делает его стандартом для хранения статических данных, резервных копий, медиафайлов и для высоконагруженных проектов.

**Ключевые особенности S3:**

- **Объектный принцип:** Файлы хранятся как объекты, каждый из которых имеет уникальный ключ (идентификатор), содержимое и метаданные (имя, размер, тип).
- **Плоская структура:** Отсутствует иерархия папок; данные организуются в "бакеты" (контейнеры), а доступ осуществляется через ключи.
- **Масштабируемость:** Неограниченный объем хранения.
- **Доступ:** Доступ к данным осуществляется через API (в основном, REST API) или графический интерфейс.
- **Совместимость:** Многие другие сервисы предоставляют S3-совместимые хранилища, используя тот же API, что и Amazon S3.

**Примеры использования:**

- Хранение изображений, видео, архивов.
- Резервное копирование данных.
- Хостинг статических веб-сайтов.
- Распределение большого объема контента (CDN).

**Протокол S3:**

- Разработан Amazon Web Services (AWS).
- Стал де-факто стандартом для объектного хранения, поэтому многие облачные провайдеры (Yandex Cloud, Cloudflare, Selectel) предлагают свои S3-совместимые решения.
