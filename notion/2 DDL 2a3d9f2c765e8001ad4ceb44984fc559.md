# 2. DDL

Категория: SQL Databases
Статус: Готово

DDL (Data Definition Language) - используется для создания или модифицирования . структур на которой базируются объекты/сущности. DDL включает в себя: **CREATE**, **ALTER**, **DROP**, and **TRUNCATE**.

### CREATE

Используется для создания таких сущностей, как БД, схем, таблиц, триггеров, индексов и т.д.

### **TRUNCATE**

- Удаляет все данные из таблицы без возможности восстановления (в отличие от DELETE).
- Нельзя использовать WHERE (а в DELETE можно).
- Быстрее, так как не логирует удаление каждой строки.
- Сбрасывает идентификаторы автоинкремента.
- Триггеры не срабатывают.

![image.png](image%20347.png)

### DROP

- Удаляет всю таблицу вместе с данными, структурой, индексами, ограничениями.
- Действие необратимо.

![image.png](image%20348.png)

## Выбор между командами

- DELETE используется, когда нужно удалить определенные записи.
- TRUNCATE используется для быстрого удаления всех данных из таблицы, сохраняя ее структуру.
- DROP используется для полного удаления таблицы из базы данных.
