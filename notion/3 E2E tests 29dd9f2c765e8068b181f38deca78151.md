# 3. E2E tests

Категория: Testing
Статус: В процессе

E2E тесты проверяют всю систему от начала до конца, имитируя действия реального пользователя.

**Что они проверяют:**

- Полные пользовательские сценарии (регистрация → вход → создание заказа → оплата)
- Работу всех компонентов вместе: фронтенд, бэкенд, БД, внешние сервисы
- Критически важные бизнес-процессы

**Для Python backend это может быть:**

- Тесты через HTTP API (полный цикл запросов)
- Тесты с использованием Selenium/Playwright для проверки веб-интерфейса
- Тесты через CLI, если это консольное приложение

**Особенности E2E:**

- Самые медленные (могут занимать минуты)
- Требуют полного окружения (все сервисы должны быть запущены)
- Хрупкие — могут падать из-за проблем в любом компоненте
- Дают максимальную уверенность в работоспособности системы
- Обычно запускаются реже (на CI/CD перед деплоем)

E2E тестов должно быть немного, только для критических сценариев. Иначе тесты будут долго выполняться и часто ломаться.
