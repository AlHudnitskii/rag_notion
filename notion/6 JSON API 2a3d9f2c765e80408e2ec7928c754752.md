# 6. JSON API

Категория: Web & Networks
Статус: Готово

JSON API - спецификация, которая определяет, как клиент и сервер должны обмениваться данными, используя формат JSON (JavaScript Object Notation). Она помогает сделать взаимодействие между разными системами более структурированным и предсказуемым, определяя правила для запросов, ответов, пагинации, фильтрации и других операций над данными.

Фундаментальное отличие JSON API начинается с использования собственного специфического типа MIME — application/vnd.api+json.
*MIME-тип (тип носителя) — это строка, которая идентифицирует формат файла, отправляемого по сети, например `image/png` для изображения в формате PNG или `text/html` для HTML-документа. Он позволяет браузерам, почтовым клиентам и другим приложениям понять, как обрабатывать и отображать полученные данные. Это стандарт для описания содержимого, который играет важную роль в таких протоколах, как HTTP и SMTP.* 

**Ключевые особенности JSON API**

- **Ресурсы**: Все данные представлены в виде объектов ресурсов, каждый из которых имеет **`type`**, **`id`** и **`attributes`** (атрибуты).
- **Атрибуты**: Содержат данные, которые относятся к конкретному ресурсу.
- **Связи (Relationships)**: Определяют отношения между ресурсами, такие как "один-к-одному", "один-ко-многим" и "многие-ко-многим".
- **Параметры запросов**: Спецификация определяет стандартные параметры для запроса данных, такие как:
    - `sort`: для сортировки данных
    - `filter`: для фильтрации ресурсов
    - `include`: для "встраивания" связанных ресурсов в ответ
    - `page`: для пагинации
- **Исключение избыточности**: JSON:API стремится избежать избыточности, предоставляя только необходимые данные и минимизируя количество запросов.
- **Согласованный формат**: Все ответы сервера имеют единообразную структуру, что облегчает обработку данных на стороне клиента.

**Преимущества JSON:API**

- **Упрощение разработки**: Стандартизированный формат облегчает взаимодействие между клиентом и сервером, снижая необходимость в документации и уменьшая количество ошибок.
- **Эффективность**: Минимизация запросов и объема передаваемых данных повышает производительность приложений.
- **Масштабируемость**: Единообразная структура данных облегчает масштабирование API и интеграцию с другими системами.
