### Unit-тестирование

Unit-тест - это автоматизированный тест, проверяющий работу отдельного “юнита” кода (функции, метода, класса) в изоляции от других частей программы.

Цели: 

1. Проверка корректности работы кода 
2. Регрессионное тестирование (новый код не ломает старый)
3. Документирование ожидаемого поведения
4. Упрощение рефакторинга

Встроенный модуль `unittest`

![image.png](image%20335.png)

Основные методы assertions:

1. assertEqual(1 + 1, 2)
2. assertNotEqual(1, 2)
3. assertTrue(1 < 2)
4. assertFalse(1 < 2)
5. assertIsNone(None)
6. assertIsNotNone("text")
7. assertIn(1, [1, 2, 3])
8. assertNotIn(4, [1, 2, 3])
9. assertRaises(ValueError): int("not a number")
10. assertIsInstance("hello", str)
11. assertRegex("hello123", r'\d+')
12. assertAlmostEqual(0.1 + 0.2, 0.3, places=7)
13. assertListEqual([1, 2, 3], [1, 2, 3])
14. assertDictEqual({'a': 1}, {'a': 1})
15. assertSetEqual({1, 2}, {2, 1})

### setUp и tearDown

1. setUp - выполняется ПЕРЕД каждым тестом
2. tearDown - выполняется ПОСЛЕ каждого теста

![image.png](image%20336.png)

Пропускать тесты можно с помощью декоратора @unittest.skip(”Причина пропуска”).
